The Root Certificate

The trust anchor for digital certificate infrastructure are the Root Certificate Authorities (CA).

A certificate authority can issue multiple certificates in the form of a tree structure. A root certificate is the top-most certificate of the tree, the private key of which is used to "sign" other certificates. All certificates immediately below the root certificate inherit the trustworthiness of the root certificate â€“ a signature by a root certificate is somewhat analogous to "notarizing" an identity in the physical world. Certificates further down the tree also depend on the trustworthiness of the intermediates (often known as "subordinate certification authorities").

For the purposes of easily creating and managing certificates at DST, this site uses a self-signed root certificate. This certificate will need to be deployed to each software system and user-agent which will rely upon the server certificates created by this Certificate Manager.

Installation Instructions

Download the root certificate. Then proceed to the following instructions depending upon how you plan to access the server. For example, if your micro-service will be accessing another micro-service whose certificate was generated by this site, then use one of the operating system specific instructions below. If you only want to access a service using a user agent, also known as a 'browser', then follow the appropriate browser installation path.

For Virtual Machines & Containers

On Linux, this process updates both the Linux system's trusted certificate authorities list AND the Java truststore, IFF Java has already been installed on the system.

Linux (Debian/Ubuntu)

Full instructions can be found here.
  1. Rename, if necessary, the certificate file so that it has a .crt file extension.
  2. Copy the root certificate .crt file to /usr/local/share/ca-certificates/.
  3. Run: sudo update-ca-certificates

Linux (Fedora/CentOS/RedHat)

Full instructions can be found here. (I've not tested this yet -- mch)
  1. Install the ca-certificates package:yum install ca-certificates
  2. Enable the dynamic CA configuration feature:update-ca-trust force-enable
  3. Add it as a new file to /etc/pki/ca-trust/source/anchors/: cp foo.crt /etc/pki/ca-trust/source/anchors/
  4. Run: update-ca-trust extract
